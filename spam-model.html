Spam message classifier (deep learning)

I built this for a university data analytics assignment. It labels SMS text as spam or not spam and outputs a probability score.

For recruiters: roles and projects are on my resume (PDF); this page is coursework. The interactive demo uses JavaScript rules for speed and static hosting. The numbers below are from the trained Keras CNN on the UCI test split, with proof in outputs/.

Approach

The model uses a convolutional neural network (CNN) on top of GloVe word embeddings (100 dimensions, trained on Twitter text). Messages are treated as sequences so the network can pick up word order and short phrases, not just a bag of words.

How it is trained

Training uses the UCI SMS Spam Collection. I used early stopping on validation loss so training stops when extra epochs stop helping, and the best weights are kept.

How it did on the test set

On a held-out test set of 1,115 messages: about 98.7% accuracy, 95.2% F1 on spam, and 99.4% ROC-AUC. Only 14 messages were wrong total.

Limitations (expected for this scope)

UCI SMS is a fixed academic set; real smishing wording changes over time. The model still has false negatives (spam marked as ham) on the test export—see outputs/test_predictions_dl.csv. High accuracy does not mean “catches every scam.”

What I used

Python, TensorFlow and Keras, NumPy, pandas, scikit-learn, and Jupyter. The embedding layer uses the GloVe Twitter 100d weights.
